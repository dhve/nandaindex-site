The basics

What is the NANDA Index, in one sentence? +

It is a trusted, tamper-proof directory that lets AI agents find each other and prove who they are, like a verified phone book for software that works on your behalf.

What is an "AI agent"? +

An AI agent is a piece of software that can do tasks for you on its own: booking an appointment, comparing prices, answering customers, or handing a job off to another agent. Think of it as a digital assistant that can take action, not just chat.

The expectation is that there will soon be enormous numbers of them, run by companies, governments and individuals, all needing to work together.

Why can't we just use what the internet already has? +

The internet's current address book was built for websites that rarely change. AI agents are different: there can be billions of them, they appear and move constantly, and you need to know whether to trust them.

The old system also only proves a name. It cannot tell you that an agent is honest, that it can really do what it claims, or instantly cut off one that turns malicious. NANDA adds those missing pieces.

Does this replace the internet? +

No. It sits on top of the internet we already have and fills one specific gap: helping agents find and trust each other. Everything else, like websites and apps, keeps working exactly as before.

Trust & safety

How does it know an agent is trustworthy? +

An agent cannot simply claim what it can do. Its identity and abilities are checked and digitally signed by a trusted source, a bit like a verified badge or a notarized document.

Anyone looking up the agent can confirm that seal for themselves, so they are not just taking the agent's word for it.

Can it be turned off if an agent goes bad? +

Yes, and quickly. If an agent is hacked or starts misbehaving, its verified status can be revoked in under a second, so others stop trusting it almost immediately.

That is a big improvement over today's systems, where cutting off trust can take hours.

Does looking something up expose my privacy? +

No. The directory is designed so you can find an agent without revealing who you are or what you are looking for. Different people can even be shown different, appropriate details.

There is also a tamper-proof record of changes, so misbehaviour can be traced without putting everyone's lookups on display.

Who is in charge of it? Is it one company? +

No single company owns it. It is decentralized: the listings are spread out and signed, and each agent or organization keeps its own details. That means it does not depend on one company staying online or behaving well.

Comparisons & more

How is this different from a normal web directory or search? +

A normal directory lists names and addresses. NANDA adds proof: it verifies who each agent is, confirms what it can do, keeps that information current, and can withdraw trust instantly. It is built for software acting on its own, not for people browsing pages.

I have heard of MCP, A2A and similar. How does this relate? +

Those are different tools that describe how agents talk to each other. NANDA is the layer underneath that handles finding and trusting them. They work together: an agent using any of those can still be listed and verified in the NANDA directory.

Can I see it or try it? +

Yes. You can browse the live directory at nandaindex.org and look up an agent's entry. Organizations can also register their agents to be listed.

Who is building this, and where can I learn more? +

It is a research effort led from MIT, called Project NANDA, with collaborators across industry and academia including Cisco and others. It is still an early, evolving project.
