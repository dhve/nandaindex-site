Glossary

Terms used across the index records, the resolution path, and the bridged discovery systems.

Groups

  1. Core model
  2. Discovery mechanisms
  3. Identity and infrastructure

Core model

NandaIndex
A federated resolution layer. An AI Catalog-formatted database of index records that maps a stable identity to the correct next discovery object.
Switchboard
The federation model: route the first resolution hop, then leave the critical path. NandaIndex bridges discovery systems rather than replacing them.
AI Catalog
A typed, nestable, machine-readable container for heterogeneous AI artifacts (A2A cards, MCP descriptors, nested catalogs, tools, skills, datasets, gateways). Media type application/ai-catalog+json.
Catalog Entry
A single record inside AI Catalog that maps an identifier to a target. NandaIndex uses the Catalog Entry as its index-record format.
Discovery object
The target an identity resolves to: a catalog, a DNS-AID record, an ADS entry, a gateway, a hosted agent card, or a sector registry.
Resolution role
A metadata hint (org.projectnanda.resolutionRole) describing how to treat the target, for example dns-aid-pointer or personal-agent-card.
Identifier (URN / URI)
The stable identity an entry resolves: urn:ai:domain:, urn:ai:email:, or urn:ai:system:.
AgentFacts
A signed, self-describing metadata document for an agent, from the NANDA Index architecture.

Discovery mechanisms

DNS-AID
DNS-based agent identity discovery. Domain-controlled, fast, cacheable resolution through DNS records, published under the _agents.<domain> naming convention (for example refunds._agents.skyblue.com).
AGNTCY
Open-source collective building the Internet of Agents (Cisco Outshift, LangChain, Galileo); a Linux Foundation project.
ADS (Agent Directory Service)
AGNTCY backend registry using IPFS Kademlia DHT routing, OCI-aligned content addressing, and Sigstore-backed integrity verification.
ARD (Agent Resource Discovery)
A read-only standard finder API across conforming registries, used the same way against systems such as Hugging Face and GitHub.
ANS
Agent Name System. Provides identity and ownership proof.
A2A Agent Card
A description of an agent's endpoint, auth, and skills. Media type application/a2a-agent-card+json.
MCP
Model Context Protocol. A server descriptor exposing tools and resources to agent runtimes.
Gateway
An access-control and policy layer in front of agents, handling privacy, throttling, and compliance.
Platform registry
A hosting platform providing onboarding, hosting, billing, and developer experience.
OASF / ACP
AGNTCY common specifications: a shared object schema and an agent connect protocol.

Identity and infrastructure

DID
Decentralized identifier. A portable, self-owned identity.
.well-known
A standard URL prefix where a domain publishes machine-readable resources such as an AI Catalog.
Permissionless deployment
Registering an identity and delegating runtime, agent card, and domain to independent providers, with no single controlling stack.
Agent card host
A third-party host that serves an agent card on behalf of an identity, separate from the runtime and the domain. host39.org is one such host, letting SMBs and individuals publish agent-card information without owning a DNS domain or running their own catalog.
Runtime
The provider hosting the executing agent (AWS, Azure, GCP, Railway, Cloudflare, and others).
DHT
Distributed hash table. The decentralized routing substrate (IPFS Kademlia) used by ADS.
OCI
Open Container Initiative. A content-addressing format reused by ADS for immutable storage.
Sigstore
A signing and integrity-verification system used by ADS.
SLIM
AGNTCY agent gateway for low-latency multi-agent communication, supporting peer-to-peer and pub/sub modes.
TTL
Time to live. The cache lifetime of a record before it is refetched.
