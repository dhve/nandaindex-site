Glossary

Terms used across the index records, the resolution path, and the bridged discovery systems.

Groups

  1. Core model
  2. Discovery mechanisms
  3. Identity and infrastructure

Core model

NandaIndex
A federated index of AI Catalog-formatted resolution records, a global switchboard for agent discovery. Each entry maps an identity to a discovery entry point. A distributed index of pointers rather than a centralized registry.
Switchboard
The global switchboard model: a federated resolution layer that maps identities to the appropriate discovery mechanism, bridging discovery systems rather than replacing them.
AI Catalog
A typed, nestable, machine-readable container for heterogeneous AI artifacts (A2A cards, MCP descriptors, nested catalogs, tools, skills, datasets, gateways). Media type application/ai-catalog+json.
Catalog Entry
A single record inside AI Catalog that maps an identifier to a target. NandaIndex uses it as its index-record format: identifier, displayName, mediaType, a url or inline data, description, tags, publisher, and metadata.
Publisher
The identity that publishes an index record, given as an identifier, displayName, and identityType (dns, email, or system).
Discovery object
The target an identity resolves to: a catalog, a DNS-AID record, an ADS entry, a gateway, a hosted agent card, or a sector registry.
Resolution role
The metadata field org.projectnanda.resolutionRole, which in the index-record examples carries values such as nested-ai-catalog, dns-aid-pointer, smb-agent-card, personal-agent-card, and ard-endpoint.
Identifier (URN / URI)
The stable identity an entry resolves: urn:ai:domain:, urn:ai:email:, or urn:ai:system:.

Discovery mechanisms

DNS-AID
DNS-based agent identity discovery. Domain-controlled, fast, cacheable resolution through DNS records, published under the _agents.<domain> naming convention (for example refunds._agents.skyblue.com).
AGNTCY
A Linux Foundation project. The global switchboard proposed by NANDA and AGNTCY was built jointly by Outshift (Cisco) and MIT.
ADS (Agent Directory Service)
AGNTCY distributed directory service implementing the discovery and infrastructure layers: content-addressed storage, distributed routing (DHT-based), and cryptographic signing and verification.
ARD (Agentic Resource Discovery)
A standard interface for querying and retrieving agentic resources across catalogs and registries.
ANS
An identity system that provides mechanisms for verifying ownership, provenance, and trust.
A2A Agent Card
A description of an agent's endpoint, auth, and skills. Media type application/a2a-agent-card+json.
MCP
Model Context Protocol. A server descriptor exposing tools and resources to agent runtimes.
Gateway
An access-control and policy layer in front of agents, handling privacy, throttling, and compliance.
Platform registry
A hosting platform providing onboarding, hosting, billing, and developer experience.

Identity and infrastructure

DID
Decentralized identifier. A portable, self-owned identity.
.well-known
A standard URL prefix where a domain publishes machine-readable resources such as an AI Catalog.
Permissionless deployment
Registering an identity and delegating runtime, agent card, and domain to independent providers, with no single controlling stack.
Agent card host
A third-party host that serves an agent card on behalf of an identity, separate from the runtime and the domain. host39.org is one such host, letting SMBs and individuals publish agent-card information without owning a DNS domain or running their own catalog.
Runtime
The provider hosting the executing agent (AWS, Azure, GCP, or another provider).
DHT
Distributed hash table. The decentralized routing substrate used by ADS for distributed routing across nodes.
TTL
Time to live. The cache lifetime of a record before it is refetched.
